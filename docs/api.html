PredictMarket API

Prediction market platform — REST API + WebSocket reference for web and admin developers.

Base URL (Dev)
http://localhost:3000
Content-Type
application/json
Currency
KES (Kenyan Shilling)
Auth Scheme
Bearer {accessToken}
Access Token TTL
15 minutes
Rate Limit
120 req/min per IP
ℹ️
All amounts are in KES (integer or float) Wallet balances, trade amounts, deposits, and withdrawals are all denominated in Kenyan Shillings. Shares are priced at KES 10/share. A 4% rake is applied on market resolution.
⚠️
Access token expiry Access tokens expire in 15 minutes. Use POST /auth/refresh with the refresh token (30-day TTL) to get a new pair. Store both tokens securely.
🔐

Authentication Flow

1
Register — POST /auth/register
Submit phone number (Kenyan format: 0712345678 or +254712345678) + password (min 8 chars). An OTP SMS is sent immediately. Returns { message, phone }.
2
Verify Phone — POST /auth/verify-phone
Submit the 6-digit OTP. Returns { accessToken, refreshToken, user }. Account is now active.
3
Login — POST /auth/login
For returning users. Returns { accessToken, refreshToken, user }.
4
Use Bearer Token
Include Authorization: Bearer {accessToken} in every authenticated request header.
5
Refresh — POST /auth/refresh
When the access token expires, call this with the refresh token. Returns a new { accessToken, refreshToken } pair. Refresh tokens are rotated on each use — store the new one.
6
Logout — POST /auth/logout
Revokes the refresh token server-side. Clear both tokens from client storage.
✅

KYC Tiers & Limits

KYC tier is embedded in the JWT payload as kycTier. Limits are enforced server-side.

Tier Status Trading Deposit / day Withdrawal / day
0 Phone verified only Allowed KES 50,000 Not allowed
1 ID submitted (pending) Allowed KES 150,000 KES 70,000 / day
2 KYC approved Allowed KES 300,000 KES 150,000 / day
ℹ️
KYC upgrade flow User submits POST /users/me/kyc/start with doc type, document number, and optional image URLs. Admin reviews via admin panel and approves/rejects. On approval tier moves to 2.

WebSocket

Connect to ws://localhost:3000 (Socket.io). Pass the access token as a query param or auth header on connect.

ℹ️
Automatic rooms On connection, the gateway automatically joins the socket to user:{userId} room for private events. You do not need to subscribe manually.
Client → Server
subscribe:market emit
Join a market's price channel. Send the market ID string.
// Connect
socket.emit('subscribe:market', 'market-uuid-here');
unsubscribe:market emit
Leave a market's price channel.
socket.emit('unsubscribe:market', 'market-uuid-here');
Server → Client
market:price receive
Real-time price update after each trade. Fires to all subscribers of market:{id} room.
{
  "marketId": "uuid",
  "yesPrice": 0.62,   // 0.00 – 1.00
  "noPrice":  0.38,
  "yesPool":  31200,  // total KES in YES pool
  "noPool":   19600,
  "updatedAt": "2024-01-15T10:30:00.000Z"
}
wallet:refetch receive
Tells client to re-fetch wallet balance. Fires on deposit confirmed or withdrawal completed. Sent to user:{userId} room only.
{
  "userId": "uuid",
  "reason": "deposit"  // "deposit" | "withdrawal" | "settlement"
}
payment:update receive
Payment status change (STK pushed, confirmed, failed). Sent to user:{userId} room only.
{
  "paymentId": "uuid",
  "type":      "deposit",   // "deposit" | "withdrawal"
  "status":    "COMPLETED", // "PENDING" | "COMPLETED" | "FAILED"
  "amountKes": 500
}
market:settled receive
When a market resolves and the user held a position. Sent to user:{userId} room only.
{
  "marketId":      "uuid",
  "marketTitle":   "Will KES/USD reach 130 by Dec 2024?",
  "winningOutcome": "YES",
  "outcome":        "YES",  // user's outcome
  "payoutKes":      1450,
  "sharesHeld":     100
}

Error Codes

All errors follow NestJS standard shape: { statusCode, message, error }.

400
Bad Request
Validation failed. message is an array of field errors from class-validator.
401
Unauthorized
Missing, expired, or invalid access token. Re-authenticate or refresh.
403
Forbidden
Authenticated but insufficient role (e.g. non-admin hitting admin route).
404
Not Found
Resource does not exist or belongs to another user.
409
Conflict
Duplicate (phone already registered, idempotency key reuse).
422
Unprocessable Entity
Business rule violation (insufficient balance, market not ACTIVE, over daily limit).
429
Too Many Requests
Rate limit exceeded (120 req/min per IP at gateway).
500
Internal Server Error
Unexpected error. Retry with exponential back-off.

Trades

Parimutuel pool trading. Shares are KES 10 each. 4% rake on resolution. Minimum trade: KES 10.

ℹ️
Idempotency Every POST /trades requires a client-generated idempotencyKey. If a request is retried with the same key, the server returns the original trade instead of creating a duplicate. Generate a UUID per trade attempt.
POST /trades auth Place a trade
Request Body
FieldTypeRules
marketId *stringUUID of an ACTIVE market
outcome *string"YES" or "NO"
amountKes *numberMin KES 10. Must be a multiple of 10 (= whole shares).
idempotencyKey *stringClient-generated UUID. Prevents duplicate trades on retry.
Response 201
{
  "tradeId":      "uuid",
  "marketId":     "uuid",
  "outcome":      "YES",
  "amountKes":    200,
  "sharesCount":  20,    // amountKes / 10
  "priceAtTrade": 0.614,  // implied prob at time of trade
  "status":       "CONFIRMED",
  "createdAt":    "2024-01-15T10:30:00.000Z"
}
Errors
422 — insufficient wallet balance; 422 — market not ACTIVE; 422 — daily limit exceeded

Payments (M-Pesa)

Deposits via STK Push. Withdrawals via B2C. All amounts in KES. Tier limits apply.

⚠️
Withdrawal requires OTP Before calling POST /payments/withdrawals/initiate, call POST /auth/request-otp with purpose: "withdrawal". Include the 6-digit OTP in the request body.
POST /payments/deposits/initiate auth Initiate STK Push deposit
Triggers an M-Pesa STK Push to the user's phone. User enters PIN on their phone. Poll /payments/deposits/:id/status or listen to payment:update WebSocket event.
Request Body
FieldTypeRules
amountKes *numberMin KES 10. Daily limits apply (KES 50k/150k/300k by tier).
phone *stringSafaricom number in Kenyan format. Can differ from registered phone.
Response 200
{
  "paymentId":   "uuid",
  "status":      "PENDING",
  "checkoutRequestId": "ws_CO_...",  // Safaricom reference
  "message":     "STK Push sent to 0712345678. Enter M-Pesa PIN."
}
GET /payments/deposits/:id/status auth Poll deposit status
Poll every 3–5 seconds after initiating. Usually resolves within 30s. Use WebSocket for real-time updates instead.
Response 200
{
  "paymentId": "uuid",
  "status":    "COMPLETED",  // PENDING | COMPLETED | FAILED | CANCELLED
  "amountKes": 500,
  "mpesaCode": "QDJ8XXXXXXX",  // M-Pesa transaction code, null if pending
  "updatedAt": "2024-01-15T10:05:30.000Z"
}
GET /payments/deposits auth Deposit history
Query Parameters
ParamType
pagenumber (default: 1)
limitnumber (default: 20)
Response 200 — paginated deposit list
POST /payments/withdrawals/initiate auth Initiate withdrawal (requires OTP)
Initiates an M-Pesa B2C transfer to the user's phone. Requires OTP sent via POST /auth/request-otp. KYC tier ≥ 1 required.
Request Body
FieldTypeRules
amountKes *numberMin KES 100. Daily limits apply (KES 70k/150k by tier).
phone *stringSafaricom number (Kenyan format)
otp *string6-digit OTP from /auth/request-otp with purpose: "withdrawal"
Response 200
{
  "paymentId": "uuid",
  "status":    "PENDING",
  "message":   "Withdrawal initiated. Funds will arrive shortly."
}
Errors
422 — insufficient balance; 422 — daily limit exceeded; 403 — KYC tier 0 cannot withdraw; 401 — invalid OTP
GET /payments/withdrawals/:id/status auth Poll withdrawal status
Response 200 — same shape as deposit status
GET /payments/withdrawals auth Withdrawal history
Response 200 — paginated withdrawal list
